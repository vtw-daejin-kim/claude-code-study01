## 🏗️ 프로젝트 전체 구조

### 프로젝트 구조 방식 제안: __Monorepo__

백엔드와 프론트엔드를 하나의 저장소에서 관리하는 방식을 추천합니다.

```javascript
genious-ecommerce/
├── backend/              # Spring Boot 백엔드
├── frontend/             # React.js 프론트엔드
├── docs/                 # 프로젝트 문서
│   ├── requirements.md
│   ├── api-spec.md
│   └── database-schema.md
├── docker-compose.yml    # 개발 환경 설정
├── .gitignore
└── README.md
```

### ⚙️ application.yml 예시

```yaml
spring:
  application:
    name: genious-backend
  
  datasource:
    url: jdbc:postgresql://localhost:5432/genious_db
    username: genious_user
    password: ${DB_PASSWORD}
    driver-class-name: org.postgresql.Driver
  
  jpa:
    hibernate:
      ddl-auto: validate
    show-sql: true
    properties:
      hibernate:
        format_sql: true
        dialect: org.hibernate.dialect.PostgreSQLDialect
  
  servlet:
    multipart:
      max-file-size: 10MB
      max-request-size: 10MB

jwt:
  secret: ${JWT_SECRET}
  access-token-validity: 3600000    # 1시간 (밀리초)
  refresh-token-validity: 1209600000 # 14일 (밀리초)

server:
  port: 8080
```

## 🗄️ 데이터베이스 (PostgreSQL)

### 버전

- __PostgreSQL__: 16.1

### Docker Compose 설정

```yaml
version: '3.8'

services:
  postgres:
    image: postgres:16.1-alpine
    container_name: genious-postgres
    environment:
      POSTGRES_DB: genious_db
      POSTGRES_USER: genious_user
      POSTGRES_PASSWORD: genious_password
    ports:
      - "5432:5432"
    volumes:
      - postgres-data:/var/lib/postgresql/data
    networks:
      - genious-network

  redis:
    image: redis:7.2-alpine
    container_name: genious-redis
    ports:
      - "6379:6379"
    networks:
      - genious-network

volumes:
  postgres-data:

networks:
  genious-network:
    driver: bridge
```
