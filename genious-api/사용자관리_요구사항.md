## 👤 사용자 관리 - 회원가입/로그인 상세 요구사항

### 1️⃣ 회원가입 (Sign Up)

#### 1.1 기본 정보 입력

__필수 입력 항목:__

- 이메일 주소 (아이디로 사용)
- 비밀번호
- 비밀번호 확인
- 이름
- 전화번호
- 약관 동의 (필수/선택)

__선택 입력 항목:__

- 생년월일
- 성별
- 마케팅 수신 동의

#### 1.2 유효성 검증

- __이메일 형식 검증__: 올바른 이메일 형식인지 확인
- __이메일 중복 검사__: 실시간으로 이미 존재하는 이메일인지 확인
- __비밀번호 강도 검증__:
    - 최소 8자 이상
    - 영문 대/소문자, 숫자, 특수문자 중 3가지 이상 조합
    - 연속된 문자/숫자 제한 (예: 123456, aaaaaa)
- __비밀번호 확인__: 입력한 두 비밀번호가 일치하는지 확인
- __전화번호 형식__: 한국 전화번호 형식 (010-xxxx-xxxx)
- __전화번호 중복 검사__: 이미 등록된 번호인지 확인

#### 1.3 이메일 인증

- 회원가입 시 인증 이메일 발송
- 인증 링크 클릭 또는 인증 코드 입력
- 인증 유효 시간: 24시간
- 미인증 계정 처리 방안:
    - __옵션 A__: 인증 전까지 로그인 불가
    - __옵션 B__: 로그인은 가능하나 일부 기능 제한

#### 1.4 소셜 로그인 연동 (선택사항)

- 구글 로그인
- 카카오 로그인
- 네이버 로그인
- 소셜 계정으로 가입 시 추가 정보 입력 (전화번호 등)

#### 1.5 약관 동의

- 서비스 이용약관 (필수)
- 개인정보 처리방침 (필수)
- 마케팅 정보 수신 동의 (선택)
- 각 약관 전체 내용 확인 가능
- "전체 동의" 체크박스 제공

---

### 2️⃣ 로그인 (Sign In)

#### 2.1 일반 로그인

__입력 항목:__
- 이메일 주소
- 비밀번호
- "로그인 상태 유지" 체크박스

__기능:__
- 이메일/비밀번호 검증
- 로그인 실패 시 명확한 오류 메시지
- 보안을 위해 "이메일 또는 비밀번호가 일치하지 않습니다" (구체적으로 어떤 정보가 틀렸는지 노출하지 않음)

#### 2.2 JWT 기반 인증

- __Access Token__:
    - 유효 시간: 1시간
    - 사용자 정보 및 권한 포함
    - 클라이언트 메모리에 저장

- __Refresh Token__:
    - 유효 시간: 14일
    - HttpOnly 쿠키 또는 Secure Storage에 저장
    - Access Token 만료 시 자동 갱신

- __Token 저장 위치__:
    - Access Token: 메모리 (state management)
    - Refresh Token: HttpOnly 쿠키 (XSS 방지)

#### 2.3 로그인 상태 유지

- "로그인 상태 유지" 체크 시 Refresh Token 유효기간 연장 (30일)
- 체크 안 할 시 브라우저 종료 시 로그아웃

#### 2.4 소셜 로그인

- OAuth 2.0 프로토콜 사용
- 각 플랫폼별 로그인 버튼
- 최초 로그인 시 추가 정보 입력 요청

#### 2.5 로그인 보안

- __연속 로그인 실패 제한__:
    - 5회 실패 시 계정 일시 잠금 (30분)
    - CAPTCHA 인증 추가

- __IP 기반 이상 접근 탐지__:
    - 다른 지역/국가에서 로그인 시 이메일 알림
    - 의심스러운 활동 감지 시 추가 인증 요구

- __동시 로그인 세션 관리__:
    - 최대 동시 로그인 세션 수 제한 (예: 3개)
    - 새 기기 로그인 시 기존 세션 확인/해제 옵션

---

### 3️⃣ 비밀번호 관리

#### 3.1 비밀번호 찾기
- 이메일 주소 입력
- 등록된 이메일로 비밀번호 재설정 링크 발송
- 링크 유효 시간: 1시간
- 새 비밀번호 입력 및 확인
- 비밀번호 히스토리 확인 (최근 3개 비밀번호는 재사용 불가)

#### 3.2 비밀번호 변경
- 로그인 후 마이페이지에서 변경 가능
- 현재 비밀번호 확인 필수
- 새 비밀번호 입력 및 확인
- 변경 완료 시 모든 세션 로그아웃 (현재 세션 제외 옵션)

---

### 4️⃣ 로그아웃
#### 4.1 로그아웃 처리
- 클라이언트의 Access Token 삭제
- 서버에서 Refresh Token 무효화
- 장바구니 등 로컬 데이터 처리 방안 결정

#### 4.2 자동 로그아웃
- 장시간 활동 없을 시 자동 로그아웃 (선택사항)
- 보안 설정에 따라 시간 설정 가능 (30분, 1시간, 2시간)

---

### 5️⃣ 사용자 세션 관리
#### 5.1 세션 정보
- 현재 로그인된 기기 목록 조회
- 각 세션 정보:
    - 기기 종류 (PC, Mobile)
    - 브라우저 정보
    - 로그인 시간
    - 마지막 활동 시간
    - IP 주소 (일부 마스킹)

#### 5.2 세션 제어
- 특정 세션 강제 로그아웃
- 모든 다른 세션 로그아웃

---

### 6️⃣ 회원 탈퇴
#### 6.1 탈퇴 프로세스
- 비밀번호 재확인
- 탈퇴 사유 선택 (선택사항)
- 주의사항 안내:
    - 진행 중인 주문 확인
    - 환불 불가 포인트/쿠폰 정보
    - 재가입 제한 기간 (예: 30일)

#### 6.2 데이터 처리
- 개인정보 즉시 삭제 또는 일정 기간 보관 후 삭제
- 주문 내역 등 법적 보관 의무 데이터는 별도 보관
- 탈퇴 후 30일간 재가입 방지

## 🤔 결정이 필요한 사항

1. __이메일 인증 필수 여부__
    - 필수로 할 것인가? (보안 강화, 가입 과정 복잡)
    - 선택으로 할 것인가? (사용자 편의성, 보안 약화)

2. __소셜 로그인 우선순위__
    - 어떤 플랫폼을 먼저 연동할 것인가?
    - MVP에 포함시킬 것인가?

3. __비밀번호 정책 강도__
    - 엄격한 정책 vs 사용자 편의성 균형

4. __동시 로그인 세션 수__
    - 제한 없음 vs 제한 (몇 개?)

5. __자동 로그아웃 기능__
    - 필요 여부 및 시간 설정
